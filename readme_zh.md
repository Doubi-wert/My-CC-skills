## 技能

### iterative-critique

双代理批判-修订循环，用于生成高质量文档和代码。

- **Generator (G)**: 基于参考文档生成内容
- **Critic (C)**: 审查内容与参考文档的一致性
- **迭代**: 持续改进直到反馈减少到阈值以下

**适用场景**:
- 迭代改进文档或代码
- 批判性审查内容是否符合参考材料
- 基于学术/技术标准进行多轮修订

> **警告**: 批判迭代涉及多轮双代理审查。这可能消耗大量 token。请在使用前确保有足够的预算。

---

### paper-learning

基于学术论文生成教学文档，采用双层批判迭代架构。内部调用 **iterative-critique** 进行内容审查和优化。

- **章节级 (Meso)**: 逐章迭代处理
- **整体级 (Macro)**: 跨章节一致性检查

**适用场景**:
- 基于论文生成教程
- 创建引用一致的教学材料
- 跨文档章节的交叉引用验证

> **输入提示**: 需要 `.md` 格式的论文内容。使用 OCR 或复制工具进行提取——但对于包含大量数学公式或图片的论文，OCR 经常出错。**建议人工审核。** **请勿直接输入 PDF；先转换为 `.md` 格式。**
